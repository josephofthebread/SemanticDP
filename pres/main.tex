\splitframe{%
  \panelhead{The question posed}
  When DP degrades LLM alignment,
  is the degradation caused by measurable semantic distortion of the training signal,
  or by other effects of DP noise -- reduced optimization quality,
  calibration shifts,
  altered refusal/helpfulness trade-offs?
}{%
  \uncover<1->{%
    We test whether alignment degradation is driven by semantic distortion or by the direct, non-semantic effects of DP noise.
    \par\vspace{0.15cm}}

  \uncover<2->{%
    Statistically,
    the naive version is a correlation question:
    does degradation track distortion?
    We reformulate it as a regression with the privacy level controlled,
    plus a randomized non-DP coefficient.
    \par\vspace{0.3cm}}

  \uncover<3->{%
    \[
      Y_{i} = a_{m_i} + b_{m_i}\, d_i + c\, \Sdist_i + e_i
    \]
    \par\vspace{0.1cm}}

  \uncover<4->{%
    \textbf{H1 (primary).}
    $|c| \ge \Delta$,
    with $\Delta = 0.10$ fixed in advance:
    semantic distortion has a \emph{practically meaningful} effect on degradation,
    not merely a nonzero one.
    \par\vspace{0.25cm}}

  \uncover<5->{%
    \textbf{H2 (secondary).}
    $|c| > |b_m|$ for every mechanism $m$:
    measured distortion predicts degradation more strongly than the nominal privacy level does.
    \par\vspace{0.25cm}}

  \uncover<6->{%
    \textbf{H3 (secondary).}
    The coefficient of $\Sdist$ is larger on the entity-rich corpus than on the entity-poor one:
    distortion hurts more when the text is fact-dense.
    \par}
}

\splitframe{%
  \panelhead{Identification}
  \textbf{Problem:} within one way of adding privacy noise,
  $\Sdist$ and $\eps$ are collinear by construction:
  more noise,
  more distortion.
  A regression on one way alone cannot separate $b_m$ from $c$.
  \vspace{0.4cm}

  We need variation in $\Sdist$ \emph{at fixed} $\eps$.
  It comes from contrasting four ways of adding (or not adding) privacy noise against each other.
}{%
  \begin{itemize}
    \item \textbf{M0 --- no privacy.}
      Standard LoRA fine-tuning on clean data;
      zero point for both $Y$ and $\Sdist$,
      the reference every other way is measured against.
    \item \textbf{M1 --- random token perturbation (non-DP control).}
      Uniform replacement/deletion at rates $p \in \{0.05, 0.15, 0.30, 0.50\}$.
      Has no $\eps$ at all,
      so it gives $\Sdist$ variation with zero collinearity to worry about.
      If it degrades alignment as much as the DP ways at equal $\Sdist$,
      the damage is semantic,
      not DP-specific.
    \item \textbf{M2 --- semantic sanitization (metric LDP).}
      SanText-style word-level sanitization with an embedding-space exponential mechanism,
      $\eps \in \{1, 3, 6, 12\}$.
      At $\eps = 1$,
      it only substitutes words with nearby embeddings -- damage is bounded by construction.
    \item \textbf{M3 --- DP-SGD.}
      Per-example gradient clipping plus Gaussian noise on the LoRA adapters,
      $\eps \in \{1, 4, 8, 16\}$.
      Perturbs gradients -- at their strictest settings, expected $\Sdist$
      differs sharply from M2's, separating $b_m$ from $c$.
  \end{itemize}
}

\splitframe{%
  \panelhead{Models and compute}
  \textbf{Models:}
  \texttt{Falcon3-1B-Instruct},
  \texttt{Llama-3.2-1B-Instruct},
  \texttt{Qwen3-1.7B}.
  The 3-4B siblings are a scale check in week 3 if compute allows.
  \vspace{0.4cm}

  3 seeds in every cell,
  so every reported point carries an error bar.
}{%
  \textbf{Models chosen for being up-to-date,
    small,
    open-weight.}
  \vspace{0.2cm}

  \textbf{The grid,
    per corpus.}
  M0 has no privacy level to vary,
  so it is counted separately:
  \begin{itemize}
    \item M0:
      $3$ models $\times$ $3$ seeds $= 9$ runs.
    \item M1--M3:
      $3$ models $\times$ $3$ mechanisms $\times$ $4$ privacy levels $\times$ $3$ seeds $= 108$ runs.
  \end{itemize}
  \vspace{0.1cm}

  $117$ runs per corpus,
  $234$ runs over the two corpora.
  \vspace{0.3cm}

  \textbf{Cost.}
  Each run is a LoRA fine-tune of a ${\le}2$B model on ${\sim}10$k instruction pairs:
  ${\approx}20$ -- $40$ GPU-minutes on one A100/H100,
  so ${\approx}80$ -- $150$ GPU-hours for the full grid.
  \vspace{0.3cm}

  \textbf{A dry run} before the full grid -- $1$ model $\times$ all ways $\times$ $1$ seed $= 13$ runs,
  under one GPU-day.
  It must show clearly different $\Sdist$ at matched privacy levels across mechanisms:
  go/no-go gate.
}

\splitframe{%
  \panelhead{Data}
  Two corpora,
  chosen to differ in \textbf{entity density}.
  \vspace{0.4cm}

  Privacy risk concentrates in names, numbers and facts,
  and mechanisms treat those tokens very differently from common words.
}{%
  \textbf{Entity-rich: Nemotron-PII}
  (\texttt{nvidia/Nemotron-PII}, CC~BY~4.0).
  $100$k synthetic records across $50+$ industries,
  with \emph{gold span-level annotations} for $55+$ PII/PHI types.
  \vspace{0.25cm}

  \textbf{Entity-poor: Alpaca}
  (a fixed 10k subset).
  Generic assistant tasks,
  few named entities,
  the standard small instruction-tuning corpus,
  which keeps our numbers comparable to prior work.
  \vspace{0.25cm}

  The contrast between the two is what makes H3 testable.
}

\splitframe{%
  \panelhead{Measuring $\Sdist$}
  $\Sdist \in [0,1]$:
  how far the meaning moved,
  computed \emph{without reference to} $\eps$.
  \vspace{0.4cm}

  The primary measure is fixed before the main pass;
  the other two only feed a sensitivity table.
}{%
  \begin{itemize}
    \item \textbf{M1--M2:}
      cosine distance between sentence embeddings of the original text and the perturbed/sanitized text.
    \item \textbf{M3:}
      DP-SGD leaves the text untouched,
      so there is no perturbed text to compare.
      We embed generations instead -- private model vs clean model,
      on the same fixed probe set of prompts -- the same cosine-distance metric,
      applied to model outputs.
  \end{itemize}
  \vspace{0.25cm}

  \textbf{Robustness:}
  \begin{itemize}
    \item \textbf{Semantic entropy} of generations -- connects $\Sdist$ to the hallucination literature at the level of meaning,
      not surface tokens.
    \item \textbf{AlignScore} -- NLI-based fact-level agreement.
  \end{itemize}
}

\splitframe{%
  \panelhead{Measuring $Y$}
  Three components,
  all \textbf{judge-free} -- no LLM judge in the measurement chain.
}{%
  \begin{itemize}
    \item \textbf{TruthfulQA} (MC1/MC2) --- truthfulness.
    \item \textbf{IFEval} --- instruction following;
      every prompt carries programmatically verifiable constraints,
      so scoring is exact.
    \item \textbf{Entity-fidelity probe} --- hallucination;
      the model reproduces facts from held-out Nemotron-PII records,
      and the answers are matched against the gold spans:
      correct,
      corrupted,
      or fabricated.
  \end{itemize}
  \vspace{0.25cm}

  TruthfulQA and IFEval are subsampled to $500$ items.
  \vspace{0.25cm}
}

\splitframe{%
  \panelhead{Analysis plan}
  One regression across the $216$ mechanism runs
  ($108$ per corpus).
  The $18$ M0 runs define the per-model baselines:
  $Y$ is the drop relative to them,
  so they are the zero point.
  \vspace{0.3cm}

  \[
    Y_{i} = a_{m_i} + b_{m_i}\, d_i + c\, \Sdist_i + e_i
  \]
}{%
  \begin{itemize}
    \item $\hat c$ with a $95\%$ CI;
      the pre-registered minimum-effect test of H1 against $\Delta = 0.10$.
    \item Per-mechanism tests of $|c| > |b_m|$,
      for H2.
    \item Mediation decomposition:
      a second regression,
      $\Sdist$ on the privacy level (per mechanism),
      gives how much the privacy level moves $\Sdist$ in the first place -- multiplied by $c$,
      that the effect,
      separated from the direct effect $b_m$.
      Confidence interval built by resampling seeds.
    \item Privacy--distortion--alignment curves:
      $Y$ against the privacy level,
      and $Y$ against $\Sdist$,
      per mechanism and per model,
      on both corpora.
    \item H3 check:
      refit allowing $c$ to differ by corpus (add an
      $\Sdist \times$ dataset term) --
      tests whether distortion hurts more on the entity-rich corpus.
    \item Diagnostics:
      confirm $\Sdist$ and the privacy level aren't too entangled to trust $b_m$ and $c$ separately,
      and that no single mechanism dominates the errors.
  \end{itemize}
}

\splitframe{%
  \panelhead{Secondary: what kind of distortion?}
  A single $\Sdist$ says how far the meaning moved,
  not \emph{what} was lost.
  \vspace{0.4cm}

  Splitting it is a secondary aim:
  it sharpens the interpretation,
  but no primary hypothesis depends on it.
}{%
  \textbf{Two components.}
  \begin{itemize}
    \item $\Sdist_{\mathrm{ent}}$ ---
      the fraction of named entities and numbers that the mechanism did not preserve,
      checked against the gold spans.
    \item $\Sdist_{\mathrm{gen}}$ ---
      the meaning drift that remains once the entities are masked out.
  \end{itemize}
  \vspace{0.25cm}

  \textbf{What we expect.}
  $\Sdist_{\mathrm{ent}}$ should drive hallucination and truthfulness,
  because those depend on getting facts right.
  $\Sdist_{\mathrm{gen}}$ should drive instruction-following,
  because that depends on intent rather than on facts.
  \vspace{0.25cm}

  Refitting the model with the two components in place of $\Sdist$,
  and reading the per-benchmark coefficients,
  tests this directly.
}

\splitframe{%
  \panelhead{Plan}
  Four weeks,
  two go/no-go gates.
  \vspace{0.4cm}
}{%
  \begin{itemize}
    \item \textbf{Wk 1.}
      Repo, environment, data pipeline,
      eval harness (TruthfulQA, IFEval, entity-fidelity probe),
      $\Sdist$ metrics unit-tested.
      \textbf{Gate: clean M0 baselines on all three models.}
    \item \textbf{Wk 2.}
      M1--M3 implemented;
      per-example-gradient smoke test on all three architectures
      (the largest technical risk, so it goes first);
      dry run $=$ 13 runs.
      \textbf{Gate: clearly different $\Sdist$ at matched privacy level;
      pre-registration commit.}
    \item \textbf{Wk 3.}
      Main pass:
      full grid on both corpora;
      curves;
      the regression and the H1--H3 tests.
    \item \textbf{Wk 4.}
      One mitigation probe at the worst cell,
      equivalence analysis if the result is null,
      paper and code release.
  \end{itemize}
}
